2. Struktur Data (Input)

2.1 article_dataset5.csv — Dataset Artikel

Kolom Tipe Deskripsi
article_id int ID unik artikel
title string Judul berita
content string Isi lengkap berita
photo_url string URL gambar thumbnail
publish_date date Tanggal terbit
url string URL asli berita di ntvnews.id
section_id string Kategori (News, Hiburan, Ekonomi, dll.)
view_count int Jumlah view artikel

2.2 acu_interactions_customized5.csv — Dataset Interaksi User

Kolom Tipe Deskripsi
interaction_id int ID unik interaksi
user_id int ID user yang melakukan klik
article_id int ID artikel yang diklik

2.3 File Turunan (Hasil Split)

File Deskripsi
train_acu_customized5.csv Data training (~80%)
test_acu_customized5.csv Data testing (~20%)

[!NOTE] Split menggunakan weighted sampling berdasarkan inverse view_count². Artikel dengan view_count rendah (unpopular) mendapat probabilitas lebih tinggi masuk ke test set, sehingga evaluasi model lebih menantang dan realistis.

3. Alur Seeding Data ke Database

Langkah 1: Seed Users → seed_users.py

Membaca data user dari CSV dan meng-insert ke tabel users di Supabase. Setiap user mendapatkan: - username dari dataset - email yang di-generate - password default (plain text = user_id)

Langkah 2: Seed Artikel & Interaksi → seed_articles_interactions.py

  1. Membaca article_dataset5.csv → Extract section_id unik → Upsert ke tabel section
  2. Insert semua artikel ke tabel article (batch 500 rows)
  3. Membaca acu_interactions_customized5.csv → Insert ke tabel user_interaction (batch 500 rows)
python seed_users.py
python seed_articles_interactions.py

4. Model Rekomendasi (Processing)

Sistem menggunakan 3 model rekomendasi yang saling melengkapi:

4.1 Popularity-Based Filtering → train_popularity.py

Aspek Detail
Algoritma Menghitung frekuensi interaksi per artikel dari data training
Input train_acu_customized5.csv + article_dataset5.csv
Proses GROUP BY article_id → COUNT → SORT DESC lalu merge dengan view_count
Output Top-K artikel terpopuler per user (eksklusi artikel yang sudah dibaca) → Tabel top_trending
Evaluasi Precision, Recall, NDCG @K=5,10,20 terhadap test set

Alur Inference (per user): 1. Hitung ranking global popularitas dari train set 2. Ambil daftar artikel yang sudah dibaca user (seen) 3. Filter out seen, ambil Top-5 → Simpan ke top_trending

4.2 Content-Based Filtering (CBF) → train_cbf.py

Aspek Detail
Algoritma TF-IDF + Cosine Similarity
Input article_dataset5.csv (kolom title + content)
Preprocessing Lowercase → Hapus URL & karakter khusus → Hapus stopwords Indonesia (tala-stopwords-indonesia.txt)
Pembobotan Title diulang 3x untuk memberi bobot lebih tinggi
Vectorizer TfidfVectorizer(max_features=10000, ngram_range=(1,2))
Output Training Matriks cosine similarity → saved_models/cbf_sim_matrix.pkl + cbf_article_info.pkl
Output Inference Top-K artikel mirip berdasarkan histori user → Tabel article_similarity

Alur Inference (per user): 1. Load matriks similarity dari .pkl 2. Ambil histori baca user dari train set 3. Untuk setiap artikel yang pernah dibaca, cari artikel paling mirip 4. Agregasi score, filter duplikat, ambil Top-5 → Simpan ke article_similarity

4.3 LightGCN (Graph Collaborative Filtering) → train_lightgcn.py

Aspek Detail
Algoritma LightGCN (Light Graph Convolutional Network) menggunakan PyTorch Geometric
Input train_acu_customized5.csv + test_acu_customized5.csv
Arsitektur Embedding Dim=8, Layers=2, menggunakan LGConv dari PyTorch Geometric
Loss Function BPR Loss (Bayesian Personalized Ranking) + L2 Regularization
Optimizer Adam (LR=1e-4, Weight Decay=5e-3)
Training Max 100 epochs, Early Stopping (patience=10), 30 steps/epoch, batch=512
Popularity Debiasing Score akhir = score - α × pop_score (α=0.1) untuk mengurangi bias popularitas
Output Training saved_models/best_model.pt + lightgcn_mappings.pkl
Output Inference Top-K rekomendasi personal → Tabel recommendation_history

Alur Training: 1. Load train CSV → Split internal 90% train / 10% validation 2. Encode user_id dan article_id ke indeks integer (LabelEncoder) 3. Bangun bipartite graph (user↔item) → edge_index 4. Training loop: BPR loss + negative sampling (popularity-weighted) 5. Early stopping berdasarkan validation loss → Simpan bobot terbaik

Alur Inference (per user): 1. Load model + mappings dari .pkl dan .pt 2. Forward pass → dapatkan user embedding & item embedding 3. Hitung dot product score untuk semua item 4. Kurangi popularity bias (score - α × pop_score) 5. Mask item yang sudah dilihat → Ambil Top-5 → Simpan ke recommendation_history

5. Pipeline Training Awal

Master Script → train_all.py

python train_all.py --epochs 100 --embedding_dim 8 --num_layers 2

Urutan Eksekusi:

Step Proses Fungsi yang dipanggil
1/4 Train CBF (TF-IDF) train_cbf.train()
2/4 Evaluasi Popularity train_popularity.train()
3/4 Train LightGCN train_lightgcn.train()
4/4 Auto-Generate Rekomendasi User 1-20 generate_recommendations.generate_for_user(uid)

Batch Inference → batch_inference.py

Untuk men-generate rekomendasi semua user sekaligus dan menyimpan hasilnya ke Supabase:

python batch_inference.py
  1. Popularity → Hitung ranking global → Generate Top-5 per user → Bulk insert ke top_trending
  2. LightGCN → Load model → Forward pass → Generate Top-5 per user → Bulk insert ke recommendation_history
  3. CBF → Load similarity matrix → Generate Top-5 similar per artikel → Bulk insert ke article_similarity

6. Pipeline User Baru (Dynamic Retraining)

Script → test_new_user_pipeline.py

Dipanggil otomatis oleh TestController.php saat user baru mendaftar melalui halaman /test.

python test_new_user_pipeline.py --user_id <id>

6 Langkah Pipeline:

Step Proses Detail
1/6 Fetch interaksi user baru Query tabel user_interaction dari Supabase untuk user_id yang baru
2/6 Gabung dengan data lama Load CSV interaksi terbaru (versi N-1), concat, deduplikasi
3/6 Update view_count artikel +1 view_count di CSV dan di Supabase untuk setiap artikel yang diklik
4/6 Train-test split Weighted split (unpopular articles → test set), filter cold-start (≥2 interaksi)
5/6 Simpan 4 CSV berversi acu_interactions_customized5(N).csv, train_...(N).csv, test_...(N).csv, article_dataset5(N).csv
6/6 Retrain & Generate Train ulang LightGCN dari epoch 0, generate Popularity + LightGCN recs untuk user baru

Sistem Versioning CSV

Setiap kali pipeline dijalankan, 4 file baru dibuat dengan suffix (N):

Versi 0 (base):  article_dataset5.csv
Versi 1:         article_dataset5(1).csv   ← view_count diperbarui
Versi 2:         article_dataset5(2).csv   ← view_count diperbarui lagi
...dst

Fungsi get_latest_article_csv() dan get_latest_interactions_csv() selalu mencari file versi tertinggi yang ada.

7. Autentikasi & Login

Controller → AuthController.php

Aspek Detail
Metode Supabase REST API (bypass PDO/IPv6 DNS issue)
Validasi Password Dual-check: Plain text (user baru dari /test) atau Bcrypt hash (user dataset)
Error Handling Pesan spesifik: "Password salah", "Email belum terdaftar", "Gagal menghubungi server"
Post-Login Set session('active_user_id') + Dispatch GenerateRecommendationsJob

Alur Login: 1. User submit email + password di /login 2. Laravel mengirim HTTP GET ke Supabase REST API: GET /rest/v1/users?email=eq.{email} 3. Jika user ditemukan → Bandingkan password (plain text atau bcrypt) 4. Jika cocok → Redirect ke /dashboard 5. Jika tidak → Tampilkan pesan error di halaman login

8. Tampilan Website (Output)

8.1 Halaman Login (/login) → login.blade.php

  • Form input email & password
  • Menampilkan pesan error validasi (blok @if($errors->any()))
  • Redirect ke /dashboard setelah login berhasil

8.2 Halaman Dashboard (/dashboard) → dashboard.blade.php

Layout utama terdiri dari 3 seksi rekomendasi:

Seksi Sumber Data Tabel Supabase Deskripsi
Berita Terkini article article 10 artikel terbaru (sort by publish_date)
Populer (Trending) Popularity Model top_trending Top-5 artikel terpopuler yang belum dibaca user
Rekomendasi Untuk Anda LightGCN Model recommendation_history Top-5 rekomendasi personal berdasarkan graph collaborative filtering

Fitur Tambahan: - Search: Pencarian artikel via query parameter ?q= - Real-time Update: Supabase Realtime listener pada tabel recommendation_history dan top_trending - Load More: Tombol untuk memuat artikel terkini tambahan - Navigasi Kategori: Link ke halaman section (News, Hiburan, Ekonomi, dll.)

8.3 Halaman Artikel (/article/{id}) → article.blade.php

  • Menampilkan detail artikel (judul, konten, gambar, tanggal, kategori)
  • Artikel Terkait (CBF): Mengambil dari tabel article_similarity → Menampilkan artikel serupa berdasarkan TF-IDF similarity
  • Tombol Bookmark: Menyimpan artikel ke tabel user_bookmarks

8.4 Halaman Kategori (/section/{id}) → section.blade.php

  • Menampilkan semua artikel dalam satu kategori (News, Hiburan, Ekonomi, Olahraga, dll.)
  • Diurutkan berdasarkan publish_date terbaru

8.5 Halaman Profil (/profile) → profile.blade.php

Tab Deskripsi
Saved Articles Artikel yang di-bookmark user (dari tabel user_bookmarks)
Reading History Semua artikel yang pernah diklik (dari tabel user_interaction), dengan pagination 9 per halaman + tombol "Lihat Histori Lainnya"
Settings Edit username, email, foto profil, password + tombol Logout

8.6 Halaman Test Pipeline (/test) → test.blade.php

Wizard 3 langkah untuk mendemonstrasikan pipeline rekomendasi:

Step Deskripsi
Step 1 Registrasi user baru (username, email, password) → Insert ke tabel users via Supabase JS Client
Step 2 Pilih artikel yang diminati (semua artikel dari Supabase) → Insert ke tabel user_interaction
Step 3 Trigger training pipeline → POST ke /test/train → TestController menjalankan test_new_user_pipeline.py → Redirect ke /dashboard

[!IMPORTANT] Alur End-to-End Ringkas: 1. CSV di-seed ke Supabase → 2. CSV di-split menjadi train/test → 3. Model Popularity + CBF + LightGCN dilatih → 4. Hasil rekomendasi disimpan ke Supabase → 5. Website Laravel membaca dari Supabase via JavaScript Client → 6. User baru mendaftar di /test → 7. Pipeline otomatis retrain + versioning CSV + update view_count → 8. Rekomendasi baru langsung tersedia di Dashboard.
