4.4 Implementasi Aplikasi Web

Subbab ini menjelaskan implementasi prototipe aplikasi web berita digital yang berfungsi sebagai proof-of-concept sistem rekomendasi anti-FOMO. Aplikasi dibangun menggunakan framework Laravel 11 sebagai backend dan Supabase (PostgreSQL cloud) sebagai basis data utama, dengan pipeline machine learning berbasis Python yang terintegrasi secara end-to-end. Arsitektur ini dipilih untuk memisahkan concern antara penyajian antarmuka (Laravel), penyimpanan data (Supabase), dan komputasi model rekomendasi (Python), sehingga masing-masing komponen dapat dikembangkan dan diuji secara independen.

4.4.1 Arsitektur Sistem dan Teknologi yang Digunakan

Bagian ini mendeskripsikan arsitektur keseluruhan sistem dan justifikasi pemilihan setiap komponen teknologi. Sistem dibangun dengan pendekatan arsitektur tiga-lapis (Three-Tier Architecture) sebagai berikut:

  1. Presentation Layer (Frontend): Laravel Blade templating engine digunakan bersama Vanilla JavaScript dan Supabase JS Client untuk real-time data fetching. Pendekatan Server-Side Rendering (SSR) via Blade dipilih agar halaman cepat dimuat, sementara data rekomendasi yang dinamis di-fetch secara asinkron oleh JavaScript setelah halaman ter-render.
  2. Application Layer (Backend): Laravel 11 menangani routing, autentikasi, session management, dan orkestrasi pipeline Python. Controller Laravel bertindak sebagai orchestrator yang memicu proses training model via popen().
  3. Data Layer: Supabase (PostgreSQL cloud) menyimpan seluruh data — users, article, section, user_interaction, user_bookmarks, serta tabel hasil rekomendasi (top_trending, recommendation_history, article_similarity).

Justifikasi Pemilihan Teknologi

  • Laravel: Dipilih karena ekosistem PHP yang matang, kemudahan routing dan session management, serta kompatibilitas dengan lingkungan hosting Indonesia.
  • Supabase: Dipilih sebagai pengganti MySQL lokal untuk mengatasi kendala koneksi IPv6 di lingkungan pengembangan (Laragon), sekaligus menyediakan fitur real-time subscription via WebSocket.
  • Python (terpisah): Ekosistem machine learning Python (PyTorch, scikit-learn, pandas) jauh lebih matang dibanding PHP, sehingga pipeline training dan inference dijalankan sebagai proses terpisah yang dipanggil oleh Laravel.

4.4.3 Alur Autentikasi dan Manajemen Sesi

Sistem autentikasi bertanggung jawab untuk mengidentifikasi pengguna dan menentukan pendekatan rekomendasi yang sesuai (cold-start vs. warm-start).

  1. Halaman Login (/login): Pengguna memasukkan email dan password. Laravel AuthController memverifikasi kredensial terhadap tabel users di Supabase via REST API. Sistem mendukung verifikasi password plain text (untuk user baru dari /register) maupun bcrypt hash (untuk user dari dataset asli). Setelah login berhasil, user_id disimpan ke session Laravel dan pengguna diarahkan ke /homepage (warm-start).
  2. Halaman Register (/register): Pengguna baru mendaftar melalui form registrasi (username, email, password, gender, usia). Data disimpan ke tabel users di Supabase via Supabase JS Client. Setelah registrasi berhasil, pengguna diarahkan ke /welcome — halaman cold-start yang hanya menampilkan rekomendasi non-personalized.
  3. Penentuan Rute Berdasarkan Status Pengguna:
    • Pengguna baru (belum memiliki interaksi) → /welcome → cold-start, rekomendasi berbasis popularitas global.
    • Pengguna lama (sudah memiliki riwayat interaksi) → /homepage → warm-start, rekomendasi personal dari LightGCN.

4.4.4 Implementasi Rekomendasi Non-Personalized (Cold-Start)

Bagian ini menjelaskan bagaimana sistem menangani cold-start problem — yaitu situasi ketika pengguna baru belum memiliki riwayat interaksi sehingga model personalized tidak dapat menghasilkan rekomendasi.

  1. Strategi Cold-Start: Popularity-Based Filtering. Ketika pengguna baru masuk melalui /register ke /welcome, sistem belum memiliki data interaksi untuk pengguna tersebut. Halaman /welcome menampilkan berita berdasarkan popularitas global (diurutkan berdasarkan view_count tertinggi) dan kronologi terbaru (publish_date descending). Pendekatan ini menggunakan prinsip wisdom of the crowd — artikel yang paling banyak dibaca oleh seluruh pengguna diasumsikan memiliki relevansi tinggi secara umum.
  2. Tampilan Halaman /welcome:
    • Hero Section: Menampilkan 1 artikel utama dengan gambar besar beserta 3 sub-artikel.
    • Sidebar Terpopuler: Top-5 artikel berdasarkan view_count tertinggi secara global.
    • Feed Terkini: Daftar artikel terbaru berdasarkan tanggal publikasi, dilengkapi gambar dan metadata (view count dengan icon mata overlay).
  3. Transisi dari Cold-Start ke Warm-Start: Setiap kali pengguna membuka halaman artikel (/article/{id}), sistem secara otomatis mencatat interaksi ke tabel user_interaction. Setelah akumulasi interaksi cukup, pengguna dapat di-retrain sehingga beralih dari cold-start ke warm-start.

4.4.5 Implementasi Rekomendasi Personalized (Warm-Start)

Bagian ini menjelaskan bagaimana sistem menyajikan rekomendasi yang dipersonalisasi untuk pengguna yang sudah memiliki riwayat interaksi.

  1. Halaman Homepage (/homepage). Setelah login, pengguna dengan riwayat interaksi diarahkan ke /homepage yang menampilkan tiga jenis rekomendasi:
    • Berita Terkini (Beranda): Artikel terbaru berdasarkan publish_date.
    • Terpopuler (Sidebar): Top-5 artikel trending per-user dari tabel top_trending — hasil Popularity-Based Filtering yang mempertimbangkan section preference pengguna.
    • Rekomendasi untuk Kamu (Sidebar): Top-5 artikel personal dari tabel recommendation_history — hasil model LightGCN.
  2. Model yang Digunakan untuk Warm-Start:
    • LightGCN (Light Graph Convolutional Network): Dipilih sebagai model personalized utama berdasarkan hasil evaluasi performa. LightGCN memodelkan relasi user-item sebagai graph bipartite dan menggunakan neighborhood aggregation untuk mempelajari embedding user dan item.
    • Content-Based Filtering (TF-IDF): Digunakan untuk fitur “Berita Serupa” di halaman detail artikel (/article/{id}), menampilkan artikel dengan konten paling mirip berdasarkan cosine similarity dari vektor TF-IDF.
  3. Real-Time Update via Supabase Channel: Frontend menggunakan Supabase Realtime (WebSocket) untuk mendengarkan perubahan pada tabel recommendation_history dan top_trending. Ketika pipeline selesai meng-insert rekomendasi baru, UI secara otomatis ter-update tanpa perlu refresh halaman.

4.4.6 Integrasi Pipeline Python–Laravel

Bagian ini menjelaskan mekanisme teknis bagaimana Laravel memanggil pipeline machine learning Python dan bagaimana hasilnya dikembalikan ke antarmuka pengguna.

  1. Arsitektur Integrasi: Laravel tidak menjalankan model ML secara langsung. Sebagai gantinya, TestController memanggil script Python (test_new_user_pipeline.py) sebagai child process menggunakan popen(). Output Python di-stream secara real-time ke frontend dalam format NDJSON (Newline-Delimited JSON), memungkinkan UI menampilkan progress training secara live.
  2. Pipeline Training untuk User Baru (Halaman /test):
    • Step 1 — Registrasi: User baru memasukkan data diri (username, email, password, gender, usia) yang disimpan ke Supabase.
    • Step 2 — Pilih Artikel: User memilih minimal 2 artikel dari katalog, interaksi disimpan ke tabel user_interaction.
    • Step 3 — Training & Inference: Laravel memicu test_new_user_pipeline.py yang menjalankan: (1) mengambil interaksi user baru dari Supabase, (2) menggabungkan dengan dataset interaksi existing, (3) re-training model LightGCN, (4) menjalankan inference Popularity, CBF, dan LightGCN, (5) menyimpan hasil rekomendasi ke Supabase.
  3. Batch Inference untuk User Existing: Script train_all.py dan batch_inference.py digunakan untuk melatih semua model sekaligus dan meng-generate rekomendasi untuk seluruh user secara batch. Script reset_and_retrain.py menyediakan tiga mode operasi: default (reset rekomendasi + retrain), --full (reset semua + seed ulang + retrain), dan --clean-only (hanya bersihkan tabel).

4.4.8 Halaman Demo Pipeline (/test)

Halaman /test dirancang khusus sebagai proof-of-concept yang memungkinkan pemangku kepentingan menyaksikan secara langsung bagaimana sistem rekomendasi bekerja — dari pembuatan user baru hingga munculnya rekomendasi personal.

  1. Alur Demo 3 Langkah (Step Wizard):
    • Step 1 — Registrasi: Form input data diri langsung terhubung ke Supabase.
    • Step 2 — Simulasi Interaksi: Pengguna memilih artikel dari katalog (searchable list dengan view count). Minimal 2 artikel harus dipilih sebagai seed interaction.
    • Step 3 — Training Live: Progress bar dan log real-time menampilkan proses training model (Popularity → CBF → LightGCN). Setelah selesai, otomatis redirect ke /homepage dengan rekomendasi personal.
  2. Nilai Demonstrasi: Menunjukkan transisi cold-start ke warm-start secara visual dan terukur, serta memberikan tangible evidence bahwa pipeline ML dapat diintegrasikan ke dalam platform berita digital existing.

4.4.9 Fitur Pendukung

Beberapa fitur tambahan melengkapi pengalaman pengguna dan mendukung pengumpulan data untuk model rekomendasi:

  1. Bookmark / Simpan Artikel: Pengguna dapat menyimpan artikel ke bookmark melalui ikon bookmark di halaman detail artikel. Artikel yang di-bookmark dapat dilihat kembali di halaman /profile pada tab “Saved Articles”.
  2. Pencatatan Histori Baca Otomatis: Setiap kali pengguna membuka halaman detail artikel, sistem secara otomatis menyimpan interaksi ke tabel user_interaction — menjadi implicit feedback yang digunakan oleh model rekomendasi.
  3. Profil Pengguna (/profile): Terdiri dari tiga tab utama: Saved Articles (grid kartu artikel yang telah di-bookmark), Reading History (daftar artikel yang pernah dibaca secara kronologis), dan Settings (form edit profil yang tersinkronisasi ke Supabase).
  4. Navigasi Kategori Dinamis: Header menampilkan link navigasi per kategori berita (Beranda, News, Hiburan, Ekonomi, Olahraga, Otomotif) yang di-generate secara dinamis dari tabel section di Supabase.
